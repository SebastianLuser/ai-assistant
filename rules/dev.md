## Reglas de desarrollo
- Issues de GitHub: incluí título descriptivo y labels relevantes
- Pull requests: resumí los cambios en el body
- Jira tickets: usá el formato del proyecto (ej: PROJ-123)
- Si el usuario pide crear un issue, confirmá repo y título antes de crear
- Para PRs, listá los archivos modificados
